#### 问题 C-001: xLSTM复杂度描述存在根本性错误

| 属性 | 内容 |
|-----|------|
| **位置** | [BHMS系统设计文档.md](file:///Users/chris/Documents/trae_projects/BHMS/Doc/BHMS系统设计文档.md#L180-L185) |
| **问题描述** | 文档声称xLSTM模块复杂度为O(L)，这是**错误**的。xLSTM中的mLSTM虽然使用矩阵记忆，但其自注意力机制的计算复杂度仍然是O(L²)，与标准Transformer相同。文档混淆了"线性复杂度"的概念。 |
| **技术影响** | 1. 架构选型依据错误，可能导致性能预期与实际严重不符<br>2. 开发者基于错误认知进行优化，浪费开发资源<br>3. 项目验收时无法达到声称的性能指标 |
| **影响范围** | 系统核心算法架构 |
| **改进建议** | 1. 修正复杂度描述：xLSTM的mLSTM模块通过矩阵记忆减少显存占用，但计算复杂度仍为O(L²)<br>2. 重新评估架构选型理由，强调显存优化而非计算复杂度优化<br>3. 补充实际性能基准测试数据支撑设计决策 |
