# BHMS 系统使用说明

更新日期：2026-03-22

本文档面向老师、答辩评审和新环境使用者，目标是在新机器上按步骤启动系统并完成一次标准演示。

## 1. 环境要求

- Python 3.11+
- Node.js 18+
- npm 9+
- Docker（仅在使用 Neo4j 图谱后端时需要）

## 2. 安装依赖

### Python 依赖

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pip install -r requirements-dev.txt
```

### 前端依赖

```bash
cd frontend
npm install
```

## 3. 启动步骤

### 步骤 1：刷新仓库默认 processed 基线

```bash
source .venv/bin/activate
python scripts/refresh_processed_baselines.py --sources nasa calce kaggle hust matr oxford pulsebat
```

这会重建 `data/processed/<source>/`，确保仓库默认生命周期训练基线是干净且可复现的。

### 步骤 2：启动 Neo4j（推荐）

```bash
docker compose -f docker-compose.neo4j.yml up -d
source .venv/bin/activate
python scripts/init_neo4j_graph.py
```

默认地址：

- Browser：`http://localhost:7474`
- Bolt：`bolt://localhost:7687`

如果现场环境不方便启动 Neo4j，可把 `BHMS_GRAPH_BACKEND` 设为 `memory`，系统仍可继续演示。

### 步骤 3：启动后端

```bash
source .venv/bin/activate
uvicorn backend.app.main:app --host 0.0.0.0 --port 8000 --reload
```

默认地址：`http://localhost:8000`

### 步骤 4：启动前端

```bash
cd frontend
npm run dev
```

默认地址：`http://localhost:3000`

## 4. 页面说明

### 首页 / 看板

用于快速说明系统目标、来源分布、整体健康状态和近期告警。

### 电池档案

用于查看电池列表、健康分、历史记录和周期数据。

### 数据导入

用于上传新样本、导入演示文件、标记训练候选，并发起“立即预测与诊断”。

### RUL 预测

用于查看：

- 预测 RUL
- 历史容量轨迹
- 寿命投影
- 特征贡献
- 时间窗口贡献

### 故障诊断

用于查看：

- 异常事件
- 候选故障排序
- 根因链
- 处理建议
- GraphRAG 图谱证据
- decision basis

### 分析中心

分析中心是答辩演示主页面，包含：

1. `RUL 分析`
2. `GraphRAG 诊断`
3. `训练与实验`
4. `数据画像`
5. `案例导出`

## 5. 标准操作流程

建议固定按下面步骤演示：

1. 上传 `data/demo_uploads/calce/calce_unseen_fault_demo.csv`
2. 点击“立即预测与诊断”
3. 在分析中心查看五个模块
4. 导出预测报告、诊断报告和案例目录

## 6. 训练与实验资产

系统已经可以展示：

- 多 seed 指标 `mean ± std`
- per-seed checkpoint 与 test details
- ablation delta
- 实验图表清单与 artifact path

如果要从当前训练池重新准备本地训练数据，可使用：

```bash
python scripts/prepare_datasets.py --source calce --include-in-training
```

## 7. 常见说明

- `data/demo_uploads` 仅用于演示上传，不会参与仓库默认 `data/processed` 基线
- 仓库已经具备多 seed / ablation 资产；当前更需要继续优化的是实验效果而不是脚本存在性
- 当前系统不以工业生产部署为目标，因此不包含企业级鉴权、监控和任务编排
