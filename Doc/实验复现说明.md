# BHMS 实验复现说明

更新日期：2026-03-15

本文档说明如何从 `data/raw`、`data/processed`、`data/models` 到案例导出，复现当前仓库的完整实验资产。

## 1. 可复现范围

当前仓库已经支持以下完整闭环：

- 三源 `data/processed` 基线重建
- `Bi-LSTM` 与 `Hybrid` 的单次训练、三 seed 汇总和来源级对比
- `Hybrid` 的 `no_xlstm`、`no_transformer`、`capacity_only` 三组消融
- 误差分布图、训练曲线、指标摘要图和 plot metadata 输出
- 案例目录导出与实验背景关联

## 2. 环境准备

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pip install -r requirements-dev.txt
```

## 3. 重建仓库默认 `data/processed` 基线

```bash
source .venv/bin/activate
python scripts/refresh_processed_baselines.py --sources nasa calce kaggle
```

该脚本只读取：

- `data/raw/nasa`
- `data/raw/calce`
- `data/raw/kaggle`

不会读取：

- SQLite 当前训练池
- `data/demo_uploads`

输出目录固定为：`data/processed/<source>/`

主要产物：

- `<source>_cycle_summary.csv`
- `<source>_split.json`
- `<source>_normalization.json`
- `<source>_feature_config.json`
- `<source>_dataset_summary.json`

## 4. 从当前训练池导出本地训练数据

如果你是在本地继续追加样本并想重新训练，可使用：

```bash
source .venv/bin/activate
python scripts/prepare_datasets.py --source calce --include-in-training
```

注意：`prepare_datasets.py` 是“本地训练池导出工具”，不是仓库默认基线生成工具。
