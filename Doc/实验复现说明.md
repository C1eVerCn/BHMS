# BHMS 实验复现说明

更新日期：2026-03-22

本文档说明如何从 `data/raw`、`data/processed`、`data/models` 到案例导出，复现当前仓库的完整实验资产。

## 1. 可复现范围

当前仓库已经支持以下完整闭环：

- 多源 `data/processed` 基线重建
- `LifecycleBiLSTM` 与 `LifecycleHybridPredictor` 的单次训练、multi-seed、ablation 和来源级对比
- 多源 `pretrain -> fine_tune` transfer benchmark
- 误差分布图、训练曲线、指标摘要图、transfer summary 和 plot metadata 输出
- case bundle 导出与实验背景关联

数据源口径固定如下：

- 主训练 / benchmark：`NASA / CALCE / Kaggle / HUST / MATR`
- trajectory 辅助源：`Oxford`
- enhancement-only 机制源：`PulseBat`

## 2. 环境准备

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pip install -r requirements-dev.txt
```

## 3. 重建仓库默认 `data/processed` 基线

```bash
source .venv/bin/activate
python scripts/refresh_processed_baselines.py --sources nasa calce kaggle hust matr oxford pulsebat
```

该脚本只读取：

- `data/raw/nasa`
- `data/raw/calce`
- `data/raw/kaggle`

不会读取：

- SQLite 当前训练池
- `data/demo_uploads`

输出目录固定为：`data/processed/<source>/`

主要产物：

- `<source>_cycle_summary.csv` 或 `<source>_cycle_summary.csv.gz`（大体量 raw-converter 训练源默认压缩）
- `<source>_split.json`
- `<source>_normalization.json`
- `<source>_feature_config.json`
- `<source>_dataset_summary.json`
- `<source>_lifecycle_dataset_summary.json`
- `<source>_lifecycle_target_config.json`

补充说明：

- `Oxford` 只提供 trajectory 辅助资产，不进入主 benchmark 排名
- `PulseBat` 只生成 enhancement asset manifest，不进入生命周期主训练

## 4. 从当前训练池导出本地训练数据

如果你是在本地继续追加样本并想重新训练，可使用：

```bash
source .venv/bin/activate
python scripts/prepare_datasets.py --source calce --include-in-training
```

注意：`prepare_datasets.py` 是“本地训练池导出工具”，不是仓库默认基线生成工具。

## 6. 多源 transfer benchmark 复现

### CALCE fine-tune

```bash
python scripts/run_transfer_benchmark.py --target calce --model hybrid --pretrain-config configs/multisource_pretrain_hybrid.yaml --finetune-config configs/transfer_calce_hybrid.yaml --seeds 7,21,42
python scripts/run_transfer_benchmark.py --target calce --model bilstm --pretrain-config configs/multisource_pretrain_bilstm.yaml --finetune-config configs/transfer_calce_bilstm.yaml --seeds 7,21,42
```

### NASA fine-tune

```bash
python scripts/run_transfer_benchmark.py --target nasa --model hybrid --pretrain-config configs/multisource_pretrain_hybrid.yaml --finetune-config configs/transfer_nasa_hybrid.yaml --seeds 7,21,42
python scripts/run_transfer_benchmark.py --target nasa --model bilstm --pretrain-config configs/multisource_pretrain_bilstm.yaml --finetune-config configs/transfer_nasa_bilstm.yaml --seeds 7,21,42
```

transfer 产物固定写入：

- `data/models/<target>/<model>/transfer/multisource_to_<target>/<model>_transfer_summary.json`
- `data/models/<target>/<model>/transfer/multisource_to_<target>/plots/`

## 7. 晋升最终 lifecycle release

```bash
python scripts/promote_lifecycle_release.py --source calce --model hybrid
python scripts/promote_lifecycle_release.py --source nasa --model hybrid
```

该命令会生成：

- `data/models/<source>/<model>/release/final_release.json`

推理端会优先使用 final release manifest；如果不存在，则回退到 transfer summary、multi-seed summary 和单次实验权重。

## 8. 封版收口与验收

如果你已经跑完实验并想把仓库整理到可复核状态，建议固定执行：

```bash
python scripts/normalize_repo_metadata_paths.py
python scripts/validate_release_assets.py
./.venv/bin/pytest -q
cd frontend && npm run build -- --outDir /tmp/bhms-frontend-build
```

其中：

- `normalize_repo_metadata_paths.py`：把 `data/models/**`、`data/processed/**` 里仍残留的仓库绝对路径改为仓库相对路径
- `validate_release_assets.py`：检查 transfer / multi-seed / ablation / comparison / final release 是否齐全，并做 8 个 `source/model` 的 release resolve smoke
- `pytest -q`：覆盖 lifecycle 预测、GraphRAG 解释、case bundle 导出等产品流 smoke
- 前端 build：确认答辩/演示页面仍可构建

## 11. 当前实验限制

- 生命周期 baseline、transfer 和 release manifest 已有脚本化流程，但这并不等于所有来源都已经达到强 benchmark 结论
- 多个来源 `R²` 仍偏弱，当前更适合写成“系统已完成实验体系与资产沉淀，但模型效果仍需继续优化”
- 元数据路径相对化和 release 资产校验已经脚本化；下一轮最重要的不是补脚本，而是继续提升 benchmark 结果质量与正式提交整理
