# BHMS 答辩演示手册

更新日期：2026-03-22

本文档用于把现场演示稳定压缩到 5-8 分钟，并保证页面、代码、论文口径一致。

## 1. 演示前检查清单

答辩前建议确认：

- 已执行 `python scripts/refresh_processed_baselines.py --sources nasa calce kaggle hust matr oxford pulsebat`
- 如使用 Neo4j，已执行：
  - `docker compose -f docker-compose.neo4j.yml up -d`
  - `python scripts/init_neo4j_graph.py`
- 后端已启动：`uvicorn backend.app.main:app --host 0.0.0.0 --port 8000 --reload`
- 前端已启动：`cd frontend && npm run dev`
- 已准备一个故障演示样本：
  - `data/demo_uploads/calce/calce_unseen_fault_demo.csv`
  - 或 `data/demo_uploads/kaggle/kaggle_unseen_fault_demo.csv`

## 2. 推荐 8 分钟演示脚本

### 第 0-1 分钟：系统定位

先讲一句总定位：

> 这是一个基于公开数据集的 BHMS 毕业设计成品基线，完成了多源导入、寿命预测、可解释诊断、实验展示与案例导出闭环，不以工业生产部署为目标。

### 第 1-2 分钟：导入未见样本

切到“数据导入”页：

1. 选择 `CALCE` 或 `Kaggle`
2. 上传 `*_unseen_fault_demo.csv`
3. 强调这是系统之前未见过的新文件
4. 点击“上传并入库”

### 第 2-3 分钟：立即预测与诊断

上传成功后点击“立即预测与诊断”，说明系统会自动完成：

- RUL 预测
- 异常检测
- GraphRAG 诊断
- 跳转到分析中心

### 第 3-4 分钟：RUL 分析

在 `RUL 分析` 中重点讲：

- 历史容量轨迹和未来寿命投影
- 关键特征贡献与关键时间窗口贡献
- 预测报告可导出

### 第 4-5 分钟：GraphRAG 诊断

在 `GraphRAG 诊断` 中展示：

- 候选故障排序
- 图谱子图
- `rule_id / source_scope / confidence_basis`
- `decision_basis`
- 诊断报告导出

### 第 5-6 分钟：训练与实验

在 `训练与实验` 中说明：

- 当前已经完成多 seed 与消融资产
- 系统能展示 `mean ± std`、per-seed、delta 和 plot 产物
- 当前真正短板不是实验脚本，而是模型效果仍需继续提升

### 第 6-7 分钟：数据画像

在 `数据画像` 中回答：

- 样本来自哪些公开数据集
- train / val / test 如何划分
- 默认训练特征是什么
- 知识库覆盖到什么程度

### 第 7-8 分钟：案例导出

在 `案例导出` 中展示：

- 最近导出目录
- 文件清单
- 自动补生成状态
- `case_bundle.md` 可作为论文附录素材

## 3. 推荐强调的亮点

- 新文件上传后可以立即完成 `导入 -> 预测 -> 诊断 -> 分析 -> 导出`
- 预测和诊断都有证据链，不是黑盒结果
- 多 seed、ablation、案例目录导出使系统超过普通演示页
- Markdown 报告与案例目录便于论文附录和现场留档

## 4. 不要夸大的内容

答辩时不要说：

- 已达到工业级部署标准
- 已在所有来源上证明 Hybrid 稳定优于 baseline
- 已解决所有实验效果问题

建议替换为：

> 当前系统已经完成了实验资产与解释链的工程闭环，但论文级说服力仍然取决于下一轮对实验效果的持续优化。

## 5. 现场兜底策略

- 如果 Neo4j 不可用：切换到 `memory` 图谱后端，说明图谱展示能力会略简化
- 如果训练页不展示最新结果：直接展示 `data/models/<source>/` 下已生成的多 seed / ablation 资产
- 如果时间不足：优先演示 `导入 -> 预测 -> 诊断 -> 案例导出`
